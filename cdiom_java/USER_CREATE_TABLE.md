## 填写说明

### 字段要求

1. **用户名 (username)**
   - 必填项
   - 长度：建议3-50个字符
   - 格式：字母、数字、下划线
   - 必须唯一，不能与现有用户重复
   - 示例：`zhang_san`, `li_si`, `warehouse_admin`

2. **手机号 (phone)**
   - 必填项
   - 格式：1[3-9]开头的11位数字
   - 必须唯一，不能与现有用户重复
   - 示例：`13800138000`, `13912345678`, `15678901234`

3. **密码 (password)**
   - 必填项
   - 建议规则：
     - 至少6个字符
     - 包含大小写字母
     - 包含数字
     - 包含特殊字符（可选）
   - 示例：`Password123`, `User@123`, `Test2024!`

4. **角色ID (roleId)**
   - 必填项
   - 可选值：1, 2, 3, 4, 5
   - 对应关系：
     - 1 = 超级管理员 (SUPER_ADMIN)
     - 2 = 仓库管理员 (WAREHOUSE_ADMIN)
     - 3 = 采购员 (PURCHASER)
     - 4 = 医护人员 (MEDICAL_STAFF)
     - 5 = 供应商 (SUPPLIER)

5. **角色名称**
   - 可选填写，用于标识
   - 对应角色ID填写对应的角色名称即可

6. **备注**
   - 可选填写
   - 用于记录用户用途、部门等信息

## 使用方式

### 方式一：通过前端界面创建
1. 使用超级管理员账号登录系统
2. 进入"用户管理"页面
3. 点击"新增用户"按钮
4. 按照表格中的信息填写表单
5. 提交创建

### 方式二：通过API创建（批量）
如果需要批量创建，可以使用以下API：

**接口地址：** `POST /api/v1/users`

**请求头：**
```
Content-Type: application/json
Authorization: Bearer {token}
```

**请求体示例：**
```json
{
  "username": "warehouse_admin",
  "phone": "13800138001",
  "password": "Warehouse@123",
  "roleId": 2
}
```

## 注意事项

1. **唯一性要求**
   - 用户名和手机号在系统中必须唯一
   - 如果创建时提示"用户名已存在"或"手机号已存在"，请修改为其他值

2. **密码安全**
   - 密码创建后会自动使用BCrypt加密存储
   - 建议使用强密码，包含大小写字母、数字和特殊字符
   - 密码不会明文显示，请妥善保管

3. **用户状态**
   - 新创建的用户默认状态为"正常"（status=1）
   - 可以直接使用创建的用户名/手机号和密码登录

4. **角色权限**
   - 不同角色拥有不同的系统权限
   - 请根据实际业务需求选择合适的角色

5. **数据验证**
   - 系统会自动验证手机号格式（必须为1[3-9]开头的11位数字）
   - 用户名不能为空
   - 密码不能为空
   - 角色ID必须有效（1-5）
